<p>For what you are trying to do, set StartBar's background to black, then you don't need to overwrite paintComponet.  </p>

<p>Don't set StartBar's bounds, use set/getPreferredSize instead.  This will allow the parent container the oppurtunity to calculate the best size for the component (which might explain your problem)</p>

<p>You seriously should conisder the use of layout managers.</p>

<p>I'd ave two child panels (content &amp; task).  I'd place all the application icons in the content, probably with a flow layout &amp; the clock as/in the task, again, probably using a flow layout.  Then I'd use either a grid bag layout or a border layout to add them to the task bar panel.</p>

<p>It might not seem like it, but ts going to make your life so much easier in the long run</p>
